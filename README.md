## `python3.13` + FAST API

#### Краткое описание проекта

Проект представляет собой API-приложение для работы с пользовательскими кошельками в разных валютах.

Пользователь может создавать кошельки, пополнять их, списывать средства, а также выполнять переводы между своими кошельками с автоматической конвертацией валют. Например, при переводе `USD -> RUB` с одного кошелька будет списано `-100 USD`, а на другой зачислено `+7500 RUB` при курсе `75`.

Доступна возможность просмотра общего баланса в одной выбранной валюте, где средства со всех кошельков будут автоматически конвертированы. Для получения актуальных коэффициентов обмена используется внешнее API. В случае недоступности внешнего сервиса приложение использует собственный резервный коэффициент с пометкой о возможной неточности.

Также реализованы:
- просмотр истории операций;
- фильтрация истории по дате;
- просмотр операций по всем кошелькам или конкретному кошельку;
- выгрузка истории операций в формате PDF за выбранный промежуток времени;
- аутентификация пользователя по логину и паролю с получением токена авторизации.

Основные отличия от исходного проекта:

- Используется **луковая архитектура (Clean Architecture)**:
  - `domain`
  - `repository`
  - `services`
  - `infrastructure`
  - `contracts/interface`

- Практически весь проект построен с использованием асинхронного подхода:
  - `postgresql + asyncpg`
  - `endpoints`
  - `pytest_asyncio`
  - `httpx`

- Реализована аутентификация и авторизация с использованием:
  - `JWT` (реализован только `access-token`);
  - хэширование паролей через `argon2`.

- Кеширование реализовано средствами Python (`dict`) без использования `Redis` и подобных решений. Вследствие этого приложение стабильнее работает при запуске с одной `instance/workers=1`.

- Добавлено декларативное развёртывание через `docker + compose`, включающее:
  - `nginx`
  - приложение
  - `postgres`

  Используется в основном для быстрого развёртывания и демонстрации проекта.

- Был использован готовый `ui/frontend/static`, который был переработан:
  - изменена разметка `index.html`;
  - добавлены дополнительные `js`-функции;
  - реализована пагинация;
  - добавлены дополнительные вызовы API;
  - добавлена кнопка скачивания PDF-файла с данными;
  - все исключения, которые возвращает backend, отображаются в интерфейсе.

  Основной фокус проекта - разработка backend-части.

- Добавлен конфигурационный модуль с настройками проекта:
  - количество `workers` при запуске;
  - количество соединений с БД;
  - время хранения кеша;
  - время жизни `access-token`;
  - настройки запросов к внешнему API;
  - Время ожидания после которого можно попробовать ещё раз сделать запрос к внешнему API;

## Запуск приложения классический подход

#### 1) Для генерации SECRET_KEY можно использовать команду:
```bash
  openssl rand -base64 48 | tr '+/' '-_' | tr -d '='
```

#### 2) Субд postgres рабочая и тестовая + асинхронный драйвер(asyncpg), можно попробовать и другую субд, но тогда: без изменения engine врят-ли получится и обязательно субд должна быть с поддержкой асинхронных запросов.

#### 3) Обязательно создаем ручками в корне проекта `/.env`:

```
DATABASE_NAME=name_db
DATABASE_USERNAME=user
DATABASE_PASSWORD=1234
DATABASE_HOST=localhost
DATABASE_PORT=5432

SECRET_KEY=create_seckret
APP_ENV=prod
```

#### 4) И для pytest в директории `/tests/.test.env` :

```
DATABASE_NAME=test_name_db
DATABASE_USERNAME=user
DATABASE_PASSWORD=1234
DATABASE_HOST=localhost
DATABASE_PORT=5432

SECRET_KEY=test_create_seckret
APP_ENV=test
```

#### 5) APP_ENV как указаны в примерах. Создаем и активируем окружение, ставим зависимости.

#### 6) Создаём/форматируем таблицы из корня проекта: `python -m app.db_table_management`

#### 7) Для генерации pdf формата используется шрифт: DejaVuSans.ttf

#### 8) запускаем из корня проекта, bash: `python main.py` или `uvicorn app.main:app --reload`, а для тестов: `pytest -vs`

#### 9) Документация по http://localhost:8000/docs а фронт по http://localhost:8000/ui/index.html - в main.py раскомментировать строку для подключения фронтенда


## Альтернативный запуск приложения через Docker/Compose; без pytest - продакшен подход

### 1) для локального запуска(на вашем пк):

- .env почти без изменения только DATABASE_HOST=На имя сервиса из docker-compose.yml
    ```
  DATABASE_NAME=name_db
  DATABASE_USERNAME=user
  DATABASE_PASSWORD=1234
  DATABASE_HOST=fastapi_psql
  DATABASE_PORT=5432
    
  SECRET_KEY=create_seckret
  APP_ENV=prod
    ``` 

- Команда для запуска:
  ```bash
  docker compose \
  -f docker-compose.yml \
  -f docker-compose.dev.yml \
  up
  ```

-  после чего localhost:8080 - фронтенд страница и так-же доступно /docs*, /api/*

- Завершить работу контейнеров:
  ```bash
  docker compose down
  ```


### 2) Запуск на vps(если у вас есть сервер с доменом и сертификатами для https):
- Скачивайте на ВАШ сервер проект любым удобным способом, заходим в корень проекта:
  - В nginx.prod.conf заменяем mysite.com на ваше доменное имя
  - Также в проекте на сервере создаётся аналогично .env(.test.env уже не нужен)
  - Создаём в корне проекта директорию /certs внутрь которой должны быть ваши сертификаты для https: fullchain.pem; privkey.pem.

- .env аналогична как в 1 случае
  ```
  DATABASE_NAME=name_db
  DATABASE_USERNAME=user
  DATABASE_PASSWORD=1234
  DATABASE_HOST=fastapi_psql
  DATABASE_PORT=5432
  
  SECRET_KEY=create_seckret
  APP_ENV=prod
  ``` 

- Команда для запуска:
  ```bash
  docker compose \
  -f docker-compose.yml \
  -f docker-compose.prod.yml \
  up
  ```
-  после чего mysite.com:443 - фронтенд страница и так-же доступно /docs*, /api/*

- Завершить работу контейнеров:
  ```bash
  docker compose down
  ```
